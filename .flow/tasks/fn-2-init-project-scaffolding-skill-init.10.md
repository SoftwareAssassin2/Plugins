## Description
Scaffold the **postgres** and **keycloak** service components (per-component docker-compose, orchestrated by `system.sh up`) and the Keycloak **realm-import JSON** for reproducible local-dev auth.

**Size:** M
**Files:** `templates/src/postgres/docker-compose.yml` (+ init); `templates/src/keycloak/docker-compose.yml`; `templates/src/keycloak/realm.template.json` (committed template — the `import/<realm>-realm.json` runtime file is generated by build-config + gitignored, NOT a committed template)

## Approach (from docs-scout/practice-scout)
- **postgres** (`src/postgres/`): `postgres` image (pin), database named **`platform`**, env from the generated `.env` (build-config). The **`owner`/`migrator`/`api` roles are bootstrapped by this component's Postgres init script** (runs before any EF migration — a migration cannot create the role it connects as); RLS policies + schema come via EF migrations (fn-2….11). Compose service launched by `system.sh up`.
- **keycloak** (`src/keycloak/`): `quay.io/keycloak/keycloak` (pin), started with **`--import-realm`**, mounting `src/keycloak/import/` at `/opt/keycloak/data/import/`. The realm **template** `src/keycloak/realm.template.json` (committed, dummy secrets, NOT in the import mount) is stamped by `build-config` (fn-2….6) into the gitignored `src/keycloak/import/<realm>-realm.json` (real secrets) — only the stamped file is mounted/imported. The realm content (R27) pre-defines: the realm; **public SPA clients** (`WebApp`, `MarketingSite` — `publicClient: true`, PKCE, no secret); a **confidential `Api` client** (`publicClient: false`); the `Api`'s **service account** (`serviceAccountsEnabled: true` on the Api client — the only backend *service* in the starter; the .NET class libraries are not services); and a baseline role set.
- **Secrets:** `src/keycloak/realm.template.json` (committed, dev-only dummy secrets) → `build-config` (fn-2….6) stamps `src/keycloak/import/<realm>-realm.json` (gitignored, mounted/imported) via deterministic JSON-field replacement of client `secret` + ids from config — never commit real secrets. Note the known `--import-realm` "only on first start / persisted-volume" caveat in keycloak.md (fn-2….8).
