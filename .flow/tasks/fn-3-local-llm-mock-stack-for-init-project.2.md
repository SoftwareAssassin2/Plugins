## Description
Wire the LLM services into the config model with an explicit build-config contract. Both `claude-api` and `openai-api` are ALWAYS present (no opt-in branch in build-config); only the local base-URL value changes on opt-in. Define exactly which `.env` file gets which env vars, update the schema doc + tests.

**Size:** M
**Files:** `templates/config.json`, `templates/config.deploy.json`, `templates/src/system-cli/build-config.sh`, `templates/docs/config-management.md`, `templates/tests/system-cli/system_cli_test.sh`, `templates/.gitignore` (add the generated `etc/local-llm/litellm/config.yaml` path ŌĆö owned by fn-2 R22, this task adds the entry).

## Approach
- Add `services.openai-api` beside `services.claude-api` in BOTH `config.json` and `config.deploy.json` (key-set parity), each with explicit `base_url` + `api_key` (never omitted). Non-opt-in local values: `claude-api.base_url=https://api.anthropic.com`, `openai-api.base_url=https://api.openai.com/v1`, keys `REPLACE_ME`. (.4's engine rewrites these to the LiteLLM values `http://127.0.0.1:4000` / `http://127.0.0.1:4000/v1` + `sk-local-mock` on opt-in.) Deploy placeholders: `{{CLAUDE_API_BASE_URL}}`, `{{CLAUDE_API_KEY}}`, `{{OPENAI_API_BASE_URL}}`, `{{OPENAI_API_KEY}}`.
- Extend `build-config.sh` (UNCONDITIONALLY) to read the always-present `base_url`+`api_key` and emit into the consuming backend component's `.env` (the `Api` by default): `ANTHROPIC_BASE_URL`, `ANTHROPIC_API_KEY`, `OPENAI_BASE_URL`, `OPENAI_API_KEY` (SDK-standard names). When embeddings are opted in, ALSO emit `OPENAI_EMBEDDING_MODEL=local-embed` (R2/R12); chat callers need no model env var (the wildcard `"*"` route serves any name). Use structured `jq --arg` reads (not string replacement). **Validate each `base_url`** against a concrete grammar with a restrictive host class (`[A-Za-z0-9.-]+` ŌĆö rejects `?#@[]:` etc. **in the host portion**; an optional path may follow; no IPv6 needed) ŌĆö `^https?://[A-Za-z0-9.-]+(:[0-9]{1,5})?(/[^[:space:]]*)?$` ŌĆö AND range-check the captured port 1ŌĆō65535 separately (the `[0-9]{1,5}` class alone accepts `99999`). An injection-resistant host char-whitelist + port-range check (NOT RFC-1123 hostname validation ŌĆö odd-but-harmless hosts like `.example.com`/`-bad` are accepted; goal is anti-injection, not URL correctness). Stricter than the permissive `v_url`; positive/negative fixtures incl. invalid + non-numeric ports + bad host chars. A malformed URL can't poison `.env`. ALL four emitted values (`ANTHROPIC_BASE_URL`/`ANTHROPIC_API_KEY`/`OPENAI_BASE_URL`/`OPENAI_API_KEY` ŌĆö `base_url` paths can still carry `#`/`$`/`"`/`'`) are emitted with an encoding **matched to the actual `.env` consumer(s)** ŌĆö FIRST identify EVERY path that reads `src/Api/.env` (investigation: shell `source`, docker-compose `env_file`, a .NET dotenv loader ŌĆö possibly more than one). Choose an encoding accepted by ALL of them, or reject chars that can't round-trip across every consumer. If shell-sourced: single-quote (`KEY='value'`, embedded `'`ŌåÆ`'\''`) and test the real source path. If a compose `env_file`/dotenv library (no shell close-reopen quoting): use that parser's escaping or **reject chars it can't represent** (embedded quotes/newlines) with a clear error. Do NOT assume shell quoting for a non-shell parser. **Split test matrix (reconciled with all-consumers rule):** each key punctuation char {space/`#`/`$`/`"`/`'`} EITHER round-trips across ALL confirmed consumers OR is rejected with a clear error (whichever the consumer set supports ŌĆö don't mandate round-trip a consumer can't represent); CR/LF/control always rejected. Base URLs grammar-validated first (spaces/malformed host/bad port rejected BEFORE encoding), then path punctuation round-tripped-or-rejected per the same rule ŌĆö do NOT test base URLs with spaces.
- **Model propagation:** `build-config` reads `localLlm` type-safely, but ŌĆö unlike `up`/`down`'s tolerant cleanup path ŌĆö **fails clearly if `.localLlm` is present and is NOT an object** (a malformed hand-edit must not silently skip stamping while the project still points at the local gateway). Genuinely-absent `localLlm` is a no-op. When `config.json` `localLlm.model` is present, `build-config` first **validates it against the Ollama model-name grammar `^[A-Za-z0-9._/-]+(:[A-Za-z0-9._-]+)?$`** (reject hostile/whitespace/metachar values ŌĆö fail clearly), then stamps the gitignored runtime `etc/local-llm/litellm/config.yaml` from `config.yaml.template` by **deterministic replacement of the named token `@@LLM_MODEL@@` with the RAW validated model name** (the template line is already `ollama_chat/@@LLM_MODEL@@` ŌĆö do NOT re-prefix `ollama_chat/`, which would yield `ollama_chat/ollama_chat/ŌĆ”`). **No `.env` is written** ŌĆö the model-pull `LLM_MODEL` (+ optional `LLM_EMBED_MODEL`) is exported by `system.sh up`/`down` from `config.json` (.3 owns that), not a build-config file. **Embeddings (R12):** the `local-embed` entry in `config.yaml.template` is wrapped in sentinel markers (`# >>> embeddings`/`# <<< embeddings`); `build-config` does a **block operation, not naive token replace** ŌĆö when `localLlm.embeddingModel` is present (validated by the same model-name grammar) it keeps the block, strips the marker lines, and substitutes `@@LLM_EMBED_MODEL@@` with the raw model; when absent it **deletes the entire marked block** (no orphan entry, no leftover `@@LLM_EMBED_MODEL@@`, valid YAML both ways). Mirrors the existing keycloak realm-stamp responsibility; absent `localLlm` ŌåÆ no-op. **Template structural validation (fail clearly, don't emit broken YAML):** beyond the missing-file case, `build-config` validates the template before/after stamping ŌĆö `@@LLM_MODEL@@` present **at least once** (it appears in BOTH the wildcard `"*"` route and the explicit `local` alias; ALL occurrences are stamped); embeddings markers (`# >>> embeddings`/`# <<< embeddings`) present, balanced, and non-duplicated; `@@LLM_EMBED_MODEL@@` present within the block when the block is kept; and **NO replacement token (`@@ŌĆ”@@`) remains in the generated `config.yaml`** (post-stamp scan ŌĆö the definitive check). Any violation ŌåÆ clear error. If `localLlm.model` is present but `config.yaml.template` is missing ŌåÆ clear error (opt-in invariant ŌĆö don't silently skip). The generated `litellm/config.yaml` is gitignored (coordinate the `.gitignore` entry with fn-2 R22).

Document the stack as dev tooling backing the services, NOT a `services{}` entry (R9).
