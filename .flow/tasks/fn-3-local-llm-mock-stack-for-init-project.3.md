## Description
Make the dispatcher forward args and orchestrate the `etc/local-llm/` stack behind the opt-in `ai`/`ai-mock` compose profiles — excluded from the default `up`, activated explicitly. The current `up.sh` ignores args, so arg-forwarding through `system.sh` + `--profile` parsing in `up.sh`/`down.sh` is an in-scope change. Cover with shell tests.

**Size:** M
**Files:** `templates/src/system-cli/system.sh`, `templates/src/system-cli/up.sh`, `templates/src/system-cli/down.sh`, `templates/tests/system-cli/system_cli_test.sh`.

## Approach
- Confirm/ensure `system.sh` forwards `"$@"` to the dispatched subcommand; update `up.sh` to parse profiles with an explicit grammar: accept ONLY `ai` and `ai-mock`, via repeatable `--profile <name>` AND `--profile=<name>` forms; reject unknown profiles/flags with usage exit 64; pass only accepted profiles through to the `etc/local-llm/` compose invocation. Default (no `--profile`) = none.
- **`LLM_MODEL` export:** `up`/`down` export `LLM_MODEL`/`LLM_EMBED_MODEL` from `config.json` via a **type-safe `jq`** — `if (.localLlm | type) == "object" then (.localLlm.model // "") else "" end` (and the equivalent for `embeddingModel`) — before invoking `docker compose`, so the compose `${LLM_MODEL:-}` / `${LLM_EMBED_MODEL:-}` interpolation (.1) reaches the model-pull container — NO `env_file`/generated `.env` (portable across compose versions). The type guard means absent OR **wrong-typed** `localLlm` (a hand-edited string/array) yields empty rather than a `jq` error — harmless for `down` and `--profile ai-mock` (which need no model; only the `--profile ai` preflight requires it non-empty). `jq` is a fn-2 devcontainer prerequisite (fn-2 R4); behavior on missing `jq`/invalid `config.json` follows fn-2's existing dispatcher convention.
- **Ambient `COMPOSE_PROFILES` sanitized:** every local-llm compose invocation runs with `COMPOSE_PROFILES=` cleared and ONLY the parsed `--profile` passed (e.g. `COMPOSE_PROFILES= docker compose -f etc/local-llm/docker-compose.yml --profile ai up`) — so an ambient `COMPOSE_PROFILES=ai` cannot start the stack on default `up` nor merge `ai` into an `ai-mock` run. Default `up` does NOT invoke the local-llm compose file at all (all services profiled → nothing to start), so the ambient var is moot there. Tested: `COMPOSE_PROFILES=ai system.sh up` (no `--profile`) starts no AI containers.
- Default `up` (no `--profile`) must NOT start the AI services. `--profile ai` → litellm(real)+ollama+model-pull; `--profile ai-mock` → litellm(mock) only. **Reject selecting BOTH `ai` and `ai-mock` on `up` (both LiteLLM instances bind `:4000` — port conflict) with usage exit 64.** Pass the accepted profile through to the `etc/local-llm/` compose invocation only; postgres/keycloak/observability behavior unchanged.
- **`down` is NOT profile-gated (resolved in review):** default `system.sh down` tears down `etc/local-llm/` if its compose file is present — invoke `docker compose -f etc/local-llm/docker-compose.yml --profile ai --profile ai-mock down` so containers started under EITHER profile are removed and nothing is left running. `down` starts nothing. `down` mirrors `up`'s profile grammar (repeatable `--profile <name>` / `--profile=<name>`, only `ai`/`ai-mock` valid; duplicates or both-together allowed since they're no-ops — `down` always tears down both internally) and rejects any OTHER arg with usage exit 64. **`down` must succeed when the generated `litellm/config.yaml` is absent** (relies on .1's exported-env + bind-mounted config + `create_host_path: false`; no `env_file`). This deliberate up/down asymmetry is tested, including the config-absent case.
- **Profile-requested-but-not-installed (resolved in review) — `up` only:** an explicit `--profile ai`/`ai-mock` to **`up`** when `etc/local-llm/docker-compose.yml` is ABSENT (non-opt-in scaffold) is a **usage error, exit 64** ("local LLM stack not installed — re-scaffold with --local-llm"). **`down` is exempt** — `down` (with or without a profile) on a not-installed stack is always a harmless no-op (teardown can't leave anything running). The `[[ -f ]]` silent-skip applies to the default `up`/`down` stack-list iteration. Tested: default `up` (no profile) non-installed = no-op; `up --profile ai` non-installed = exit 64; `down --profile ai` non-installed = no-op (exit 0).
- **`up --profile ai` preflight (stack installed):** require the generated `etc/local-llm/litellm/config.yaml` present AND `config.json` `localLlm.model` non-empty; if missing, exit non-zero with a clear "run `build-config` first" message (don't let a bind mount silently create a dir — paired with .1's `create_host_path: false`). **Stale-config guard (R12) — both directions, chat + embeddings:** the generated `config.yaml` must match current `config.json`, else `up --profile ai` would pull one model while LiteLLM serves another. Fail with "config is stale — rerun `build-config`" when: (a) ANY stamped chat-route model in `config.yaml` ≠ `localLlm.model` — the real template has TWO chat routes (wildcard `"*"` AND explicit `local`), so check ALL `ollama_chat/<x>` lines, not just one; (b) `config.yaml` has a `local-embed` entry but `localLlm.embeddingModel` is empty/missing; (c) `localLlm.embeddingModel` is set but `config.yaml` has no `local-embed` entry; OR (d) the stamped `local-embed` model in `config.yaml` ≠ `localLlm.embeddingModel` (stale embedding VALUE, e.g. config says `mxbai-embed-large` but YAML still has `nomic-embed-text`). (Extract the stamped model(s) from the deterministic generated YAML via an **anchored, fixed-string** parse — e.g. read ALL `model: ollama_chat/<x>` chat lines (wildcard + `local`) and the `model: ollama/<x>` embed line, strip the known prefix — then compare as literal strings, NOT a loose regex match (model names contain `.`/`/`/`:` and would mis-match). Tested with model names containing `.`/`/`/`:` and stale old≠new values.) `--profile ai-mock` is exempt (static config, no model).
- **Preflight ordering (resolved in review):** arg parsing + ALL `ai` preflights (not-installed → exit 64; config.yaml present; staleness; embeddings consistency) run **before any `docker compose up`** — a failed `--profile ai` must NOT leave postgres/keycloak/observability partially started. Tested: zero `docker` invocations when `--profile ai` fails preflight (stub `docker` and assert it's never called).
- Keep the `[[ -f "$stack" ]]` guard so a not-installed stack is a no-op. Update `# Description:` comments to mention the opt-in local-llm stack.
- Mind gotchas: profiled service not auto-started by another's `depends_on`; shared network; `condition: service_healthy` on model-pull.
